# 修改说明

本文档说明项目中的代码修改：**修改哪个文件、哪一行、改什么、为什么**。

---

## 一、Validate 标准化（最新）

### 新增：`utils/validator.py`

可复用的校验模块，包含：
- `validate_pos_data(path)`：校验 _pos_data.json
- `validate_payload(path)`：校验 _payload.json
- 常量：`EXIT_OK`、`EXIT_FAIL`、`REQUIRED_PAYLOAD_FIELDS`、`PAYLOAD_LIST_FIELDS`

**作用：** 校验逻辑集中，可被 scripts/validate.py 和测试代码 import 复用。

### 重构：`scripts/validate.py`

- 使用 `utils.validator` 中的函数，脚本只负责：运行 main.py → 调用校验 → 输出结果
- 常量统一命名：`POS_DATA_PATH`、`PAYLOAD_PATH`
- 抽出 `run_main()` 函数，结构更清晰

### 修改 4：第 36-42 行

**原代码：**
```python
    if not data.get("Transactions"):
        print("Transactions is empty or missing")
        return 1

    if data.get("ProcessingStatus") != "Success":
        print(f"ProcessingStatus is not Success: {data.get('ProcessingStatus')}")
        return 1

    print("Validation OK")
    return 0
```

**修改为：**
```python
    if not pos_data.get("Transactions"):
        print("Transactions is empty or missing")
        return 1

    if pos_data.get("ProcessingStatus") != "Success":
        print(f"ProcessingStatus is not Success: {pos_data.get('ProcessingStatus')}")
        return 1

    # Validate _payload.json
    if not EXPECTED_PAYLOAD.exists():
        print(f"Payload not found: {EXPECTED_PAYLOAD}")
        return 1

    try:
        payload = json.loads(EXPECTED_PAYLOAD.read_text())
    except json.JSONDecodeError as e:
        print(f"Invalid _payload.json: {e}")
        return 1

    for key in ("Id", "StoreId", "TotalAmount"):
        if key not in payload:
            print(f"_payload.json missing required field: {key}")
            return 1

    if not isinstance(payload.get("SalesLines"), list):
        print("_payload.json SalesLines must be a list")
        return 1
    if not isinstance(payload.get("TenderLines"), list):
        print("_payload.json TenderLines must be a list")
        return 1

    print("Validation OK")
    return 0
```

**为什么要改：**  
1. 把 `data` 改为 `pos_data`，与前面变量一致。  
2. 增加对 `_payload.json` 的校验，保证 CI 同时检查 pos_data 和 payload。  
3. 检查 payload 必填字段和结构：`Id`、`StoreId`、`TotalAmount`，以及 `SalesLines`、`TenderLines` 为 list。

## 三、`utils/payload_builder.py`

该文件为整体补全，不是单行小改。按功能分段说明如下。

### 1. 第 1-10 行：文档字符串、导入、常量

```python
"""Convert _pos_data.json structure to _payload.json format for downstream API."""

import uuid
from datetime import datetime


DEFAULT_CHANNEL_ID = 5637192654
DEFAULT_PRODUCT_ID = 5637146394
DEFAULT_ITEM_ID = "000006056"
STORE_ID_PREFIX = "ac4_"
```

**作用：**  
- `uuid`：生成 `InternalTransactionId`  
- `datetime`：把日期时间转成 Unix 时间戳  
- 常量：供 payload 和 SalesLine 使用

---

### 2. 第 13-69 行：工具函数

| 函数 | 作用 |
|------|------|
| `_float(s, default)` | 安全转 float，失败时返回 default |
| `_receipt_time_to_seconds(time_str)` | `"HH:MM:SS"` → 当天秒数 |
| `_date_time_to_unix(date_str, time_str)` | `"YYYY-MM-DD"` + `"HH:MM:SS"` → Unix 时间戳 |
| `_ext_prop_str(key, val)` | 构造 `ExtensionProperties` 的 StringValue 项 |
| `_ext_prop_decimal(key, val)` | 同上，DecimalValue |
| `_ext_prop_int(key, val)` | 同上，IntegerValue |

**作用：** 统一类型转换和扩展属性构造，避免重复代码和类型错误。

---

### 3. 第 72-185 行：`_build_sales_line_from_fuel`

**作用：** 把 `_pos_data.json` 中每条 `fuel_lines` 转成一条 `SalesLine`，并填充 ExtensionProperties（FuelGradeId、FuelPositionId、LineType=4 等）。

**如需改：**  
- 映射字段：改第 77-84 行及 return 里的字段  
- 扩展属性：改 `ext_props` 列表  
- 默认值：改 `DEFAULT_PRODUCT_ID`、`DEFAULT_ITEM_ID` 等

---

### 4. 第 188-240 行：`_build_tender_line`

**作用：** 把 `tender_lines` 转成 `TenderLine`，含 ExtensionProperties（TenderSubCode 等）。

**如需改：**  
- 支付相关字段：改 return 里的 Amount、TenderTypeId、Currency 等  
- 扩展属性：改 `ext_props` 列表

---

### 5. 第 243-350 行：`build_payload`

**作用：** 从 `pos_data` 取第一条交易，构造完整 payload：  
- 第 253-262 行：从 tx 取出 store_loc、register_id、total_amount、时间等  
- 第 264-283 行：构造交易层 `ExtensionProperties`（BullochShiftId、ReceiptDate、ReceiptTime、TransactionTotal*、CashierId 等）  
- 第 285-294 行：调用上面两个函数生成 `sales_lines`、`tender_lines`  
- 第 296-348 行：拼出 payload 对象，写入 `SalesLines`、`TenderLines`、`ExtensionProperties`

**如需改：**  
- 交易层扩展：改第 264-283 行的 `ext_props`  
- 顶层字段：改 payload 字典里的字段  
- 业务日期/时间格式：改第 262 行的 `transaction_time_unix` 或 BusinessDate 的格式
